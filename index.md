1. Introduction

This Privacy Policy explains how Image to PDF (“the App”) handles user information. The App is designed primarily as an offline-first utility that allows users to create PDF files from images and use related on-device features such as OCR, highlighting, quality checks, and optional voice-note attachment.

We are committed to handling user data transparently and responsibly. This Policy describes what information the App accesses, how it is used, whether it is shared, and what choices users have.

2. Summary of Data Practices

Image to PDF is designed to process most user content locally on the user’s device.

In general:

Images selected by the user are processed on the device to create PDF files.

OCR text extraction is intended to run locally on the device.

Optional voice notes are recorded only when the user chooses to use that feature.

Generated PDFs, OCR text files, thumbnails, and optional audio files are stored locally on the device unless the user explicitly shares or exports them.

We do not sell user data.

We do not collect personal files, microphone recordings, or document contents to our own servers unless explicitly stated otherwise in a future version of the App.

3. Information the App May Access

Depending on the features used, the App may access the following categories of data:

3.1 Images and documents selected by the user

The App accesses images chosen by the user in order to:

create PDF files,

arrange pages,

run OCR,

perform document-quality checks,

generate previews and thumbnails,

support markup and highlighting features.

3.2 Microphone / audio recordings

If the user chooses to record a voice note, the App requests microphone access in order to record audio attached to a PDF bundle.

Microphone access is:

optional,

requested only when the user starts the voice-note feature,

not required for basic image-to-PDF conversion.

3.3 OCR text output

When OCR features are used, the App may generate text output extracted from document images. This OCR text may be saved locally on the device to support searchable content, naming suggestions, previews, or related document features.

3.4 App usage and technical information

The App may use standard platform or SDK services necessary for app functionality, security, monetization, crash prevention, subscription handling, or consent handling, depending on the version of the App. Such services may include advertising, billing, or consent SDKs integrated through Android or Google Play components.

4. How We Use Information

The App uses accessed information only for legitimate app functions, including:

converting selected images into PDF files,

improving document readability through local processing,

extracting text using OCR,

suggesting names or organizing files,

saving user-created PDF bundles and related local assets,

recording optional voice notes linked to a document,

enabling highlighting or markup workflows,

supporting app monetization features such as ads or premium subscriptions where applicable,

maintaining app stability, compatibility, and security.

We do not use user document content or microphone recordings for unrelated profiling or sale.

5. Data Storage

Image to PDF is designed as an offline-first application.

User-generated content may be stored locally on the device, including:

PDF files,

OCR text files,

thumbnails,

optional audio recordings,

optional markup/highlight metadata,

local app database records used to manage files and history.

This data remains on the user’s device unless the user chooses to export, share, back up, or otherwise transmit it using device or third-party services.

6. Data Sharing

We do not sell personal data.

We do not share the contents of user images, OCR text, PDFs, or voice recordings with third parties except in the following situations:

when the user explicitly chooses to share or export a file,

when required for lawful compliance,

when limited technical data is processed by third-party SDKs or platform services used in the App, such as advertising, billing, consent, analytics, or security-related components, if enabled in the distributed version of the App.

If advertising or subscription services are enabled, those services may process limited technical or device-related information according to their own policies.

7. Permissions Used by the App

The App may request the following permissions depending on the features present in the installed version:

7.1 Microphone (RECORD_AUDIO)

Used only for the optional voice-note feature.

Purpose:

to let the user record an audio note linked to a PDF bundle.

This permission is not required for core PDF creation. If the user does not grant this permission, the voice-note feature will not work, but other core app functions may continue to operate.

7.2 Internet / network-related permissions

If present, these permissions may be used for:

loading advertising,

validating subscriptions or purchases,

obtaining consent settings where applicable,

downloading optional OCR language or feature components through supported delivery mechanisms,

essential app service communication.

7.3 File and media access

The App accesses user-selected images, documents, and output locations only as needed to perform requested operations such as importing, exporting, saving, opening, or sharing files.

9. Children’s Privacy

The App is not directed specifically to children under the age required by applicable law to provide valid consent in their jurisdiction. We do not knowingly collect personal information from children through our own servers as part of the App’s primary offline-first workflow.

If you believe a child has provided personal information in a way that creates a privacy concern, please contact us using the details below.

10. Data Retention

Because the App is designed primarily for local use, user-created files and related content remain on the user’s device until the user deletes them, uninstalls the App, clears app data, or removes the files manually.

Technical or transaction-related records associated with advertising, billing, or platform services may be retained according to the practices of those respective providers.

11. User Choices and Controls

Users may:

choose whether to record a voice note,

deny microphone permission,

delete generated PDFs, OCR outputs, audio files, or related content from within the App or device storage,

control app permissions in Android settings,

uninstall the App at any time.

On Android, microphone access is a runtime permission and users can revoke it later through system settings.
